# Repository Guidance

## Worklog Discipline

- Update `worklog.md` for every repository change.
- Before editing `worklog.md`, review the current entry for today's date.
- Group all changes by day using a date heading in `YYYY-MM-DD` format.
- Add new notes under today's heading instead of creating duplicate headings.
- Keep entries concise and focused on what changed, why it changed, and any important verification.
- If today's heading does not exist, create it before adding notes.
